# Features

## Settings
- Add new field types (string, link, bool, number, ref)
- Add parameters for each field types (ex: ref => entity types, entity bundles)
- Set global parameters (ex: Limit)
- Every parameters at component-side will be a input of type string
- Being able to export and import settings (is a hash good enough ?)

## Component (root)
- For each component
  - A title
  - Some fields
  - Parameters for the fields
- Being able to add multiple components in one page
- Being able to export the components settings
  - In a URL
  - As a Json
  - As a hash ?
